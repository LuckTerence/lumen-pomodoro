## 1. 文档信息

| 项目 | 内容 |
|------|------|
| 产品名称 | Lumen Pomodoro（灵动岛式专注番茄钟） |
| 产品定位 | 顶栏胶囊主交互的桌面专注计时工具：好看、顺手、切窗口不丢 |
| 运行平台 | Windows 10/11 + macOS |
| 文档类型 | PRD 产品需求文档 |
| 当前版本 | V3.0（方案 B：灵动岛优先） |
| 目标用户 | 备考/自习/写作/编程等需要长时间专注的个人用户 |

| 实际技术栈 | Win：.NET 8 + WPF + WPF-UI；Mac：Swift + SwiftUI |

---

## 2. 产品背景

用户在桌面上长时间专注时，传统番茄钟有两个痛点：

1. **一切走就丢**：倒计时困在主窗口里，切到笔记/浏览器/IDE 后无法一眼看到进度，也无法快速暂停。
2. **提醒容易被淹没**：声音被静音、弹窗被挡、系统通知进勿扰；手机提醒又容易刷飞。

Lumen 的解法不是「再加一个更吵的提醒」，而是把**主交互**做成类似苹果「灵动岛」的顶栏胶囊：

- 随时可见倒计时（Compact）
- 点一下即可暂停 / 继续 / 选任务 / 开专注（Expanded）
- 完成、预告、走神等事件短暂弹出（Transient）

**摄像头指示灯**仅作为**高级可选增强**（默认关闭）：需要更强休息提示的用户可在设置中开启。开启后不拍照、不录像、不上传。

产品原则：**好看 + 顺手** 优先于功能堆叠；主路径应能在 15 秒内演示清楚。

---

## 3. 产品目标

### 3.1 核心目标

- 顶栏灵动岛为**默认主交互**；用户几乎不必常开主窗口。
- 界面简约、耐看；计时、暂停、切任务路径尽量短。
- 支持轻量任务分类与学习统计（够用即可，不做复杂任务管理）。
- 支持弹窗、声音、系统通知等辅助提醒，可单独开关。
- **不自动进入休息，不自动循环**，关键流程由用户手动确认。
- 支持托盘（Win）/ 菜单栏（Mac）与可选开机自启。
- 本地优先：无账号、无云同步；数据可导出。
- 摄像头灯为高级选项：默认关；若开启则本地调用、无存储、无上传。

### 3.2 非目标（明确不做）

- 不做用户登录。
- 不做云同步。
- 不做人脸识别。
- 不拍照、不录像、不上传摄像头画面。
- 不做团队协作。
- 不做复杂社交、排行榜、打卡社区。
- 不做过度复杂的任务管理系统。
- **不以摄像头灯为默认主卖点**（避免与灵动岛叙事冲突）。

### 3.3 反需求（用户可能想要但明确决定不做）

| 反需求 | 不做理由 |
|--------|----------|
| 番茄钟自动循环 | 备考场景需要手动确认：用户可能在收尾、记笔记、换科目 |
| 自动进入休息 | 同上，尊重用户节奏 |
| 手机端同步 | 隐私优先 + 本地优先；手机提醒反而容易刷手机 |
| AI 学习建议 | 过度设计；核心价值是「看得见、控得住」的计时壳 |
| 云端数据备份 | 本地 JSON 已足够；用户可自行拷贝数据目录 |
| 社区/排行榜/打卡分享 | 学习是自我对抗，不需要社交压力 |
| 默认强制开摄像头灯 | 权限成本高、与「好看顺手」主路径冲突；仅高级可选 |

## 4. 用户定位

### 4.1 主要用户

备考、自习、写作、编程等任何需要长时间专注的个人场景；考研科目模板为可选便利，不绑定唯一身份。

### 4.2 用户特点

- 使用 Windows 或 macOS 桌面学习 / 办公，常在多窗口之间切换。
- 希望**不盯着番茄钟主窗口**也能看见进度、随时暂停。
- 希望界面简约、高级、耐看（对标 TomatoBar / Pomotroid）。
- 需要按科目/类别轻量记录专注任务，并看到每日/每周统计。
- 部分用户需要更强休息提醒时，再开启摄像头灯等高级选项。

## 5. 产品体验风格

### 5.1 视觉风格

界面风格参考 macOS 的简约、大气、通透感，同时适配 Windows 桌面环境。

关键词：

- 简约
- 大气
- 玻璃拟态 / Acrylic
- 柔和阴影
- 半透明面板
- 细腻圆角
- 轻量层级
- 不堆砌按钮

### 5.2 Windows 实现

- WPF-UI 框架 + 自定义 Acrylic 效果
- 半透明卡片 (`#3327272A` 等)
- 极光渐变（蓝色→翡翠绿）
- 圆角窗口
- 平滑过渡动画
- 深浅色主题自适应
- Inter 字体

### 5.3 动效要求

#### 必须动效

| 场景 | 动效要求 |
|------|----------|
| 打开软件 | 主面板轻微淡入 + 上浮 |
| 开始专注 | 倒计时状态平滑切换，按钮状态过渡 |
| 暂停专注 | 时间数字轻微缩放或透明度变化 |
| 专注完成 | 主计时区域出现柔和呼吸动画 |
| 摄像头提醒中 | 摄像头状态点或光环持续呼吸 |
| 进入休息 | 页面状态从专注卡片平滑切换到休息卡片 |
| 托盘恢复窗口 | 窗口淡入恢复 |

#### 禁止动效

- 禁止大幅抖动
- 禁止廉价闪烁
- 禁止满屏粒子特效
- 禁止影响操作响应速度的复杂动画
- 禁止让倒计时数字每秒乱跳造成视觉疲劳

## 6. 核心用户流程

### 6.1 专注流程

```
选择任务 / 科目
↓
设置或使用默认专注时间（可 ±5 分钟微调）
↓
点击"开始专注"
↓
进入专注倒计时（走神检测在后台运行）
↓
倒计时结束
↓
触发摄像头灯提醒 + 可选弹窗 / 声音 / 系统通知
↓
显示专注完成面板：评分 + 笔记 + 休息选择
↓
用户手动评分（可选）+ 写笔记（可选）
↓
用户手动选择：短休息 / 长休息 / 跳过休息
```

### 6.2 休息流程

```
专注结束 → 用户点击"开始休息"（短休息 / 长休息）
↓
进入休息倒计时
↓
（跟随休息模式下，摄像头灯持续亮起）
↓
休息结束 → 弹窗/通知提示
↓
用户手动开始下一轮
```

### 6.3 不自动流程原则

本产品不自动进入休息倒计时，也不自动开启下一轮番茄钟。

原因：
- 用户可能刚好还在收尾一道题。
- 用户可能需要先记录错题或标注进度。
- 用户可能需要决定是否继续同一科目。
- 考研场景不适合完全自动循环，容易和真实学习节奏冲突。

## 7. 核心功能需求

### 7.1 番茄钟计时

**功能说明：** 用户可以启动一个专注倒计时。

**功能点：**
- 默认专注时间：25 分钟
- 支持自定义专注时间（1-120 分钟）
- 支持开始、暂停、继续、重置计时
- 倒计时过程中展示剩余时间和圆环进度
- 当前专注任务需要和本轮番茄钟绑定
- 空闲状态支持 ±5 分钟快速调整

**验收标准：**
- 用户点击"开始专注"后，倒计时每秒递减
- 用户点击"暂停"后，倒计时停止
- 用户点击"继续"后，从暂停时间继续倒计时
- 用户点击"重置"后，倒计时恢复默认时间，当前 session 不记录
- 专注完成后，系统记录本轮完成数据（任务、时长、时间戳）

---

### 7.2 摄像头灯提醒

**功能说明：** 当专注时间结束后，程序调用本机摄像头，使摄像头指示灯亮起，用于提醒用户休息。

#### 亮灯模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 固定时长模式 | 摄像头灯持续亮起指定时长 | 用户只需要明显提醒，不希望长时间占用摄像头 |
| 直到确认模式（默认） | 摄像头灯持续亮起，直到用户点击确认 | 防止用户错过提醒 |
| 跟随休息模式 | 摄像头灯在休息倒计时期间保持亮起 | 用户希望摄像头灯作为完整休息状态提示 |

#### 摄像头提醒强度等级

| 强度 | 行为 |
|------|------|
| 轻量（Light） | 仅亮灯，不做额外动作 |
| 中度（Medium，默认） | 亮灯 + 摄像头指示灯窗口显示 |
| 严重（Severe） | 亮灯 + 主窗口强制置顶3秒 + 走神检测触发时强制置顶 |

#### 摄像头提醒配置项

| 配置项 | 类型 | 默认值 |
|--------|------|--------|
| 摄像头提醒开关 | Boolean | 开启 |
| 摄像头提醒模式 | Enum | 直到确认模式 |
| 固定亮灯时长 | Number | 3 分钟 |
| 休息期间是否亮灯 | Boolean | 开启 |
| 摄像头编号 | Number | 0 |
| 提醒强度等级 | Enum | 中度 |
| 提醒中允许手动关闭 | Boolean | 开启 |

#### 功能点
- 到点后根据用户选择的模式调用摄像头
- 摄像头提醒模式必须可在设置中修改
- 摄像头亮灯持续时间必须可配置
- 摄像头提醒期间，界面应展示当前提醒状态
- 用户可以手动停止摄像头提醒
- 摄像头调用结束后必须自动释放
- 摄像头打开失败时，弹出错误提示并引导到 Windows 隐私设置
- 软件**不得保存、展示、上传摄像头画面**
- 摄像头连续运行超过 30 分钟自动保护释放

---

### 7.3 到点提醒（多重兜底）

**功能说明：** 番茄钟结束时，除了摄像头灯提醒，也提供基础提醒方式，避免摄像头不可用时完全无提示。

| 提醒方式 | 默认状态 | 说明 |
|----------|----------|------|
| 摄像头灯提醒 | 开启 | 核心提醒方式 |
| 弹窗提醒 | 开启 | 到点后弹出完成面板 |
| 声音提醒 | 开启 | 播放提示音 |
| 系统通知 | 开启 | 使用 Windows 通知中心提醒 |
| 灵动岛通知 | 开启 | 窗口最小化时在屏幕角落显示浮动倒计时 |

**验收标准：**
- 用户可以独立开启或关闭每一种提醒方式
- 如果用户关闭所有提醒方式，系统提示风险
- 摄像头打开失败时，使用弹窗、声音或系统通知兜底
- 灵动岛仅在主窗口最小化/非激活时显示

---

### 7.4 手动休息倒计时

**功能说明：** 专注结束后，系统不自动进入休息阶段。用户需要点击"开始休息"后才进入休息倒计时。

**功能点：**
- 默认短休息时间：5 分钟
- 默认长休息时间：15 分钟
- 长休息间隔：每完成 4 个番茄钟后，系统高亮推荐长休息
- 支持用户手动开始休息（短休息/长休息）
- 支持用户跳过休息
- 支持用户提前结束休息
- 支持休息结束提醒
- 支持休息阶段是否联动摄像头灯

**验收标准：**
- 专注结束后，系统停留在"专注完成"状态
- 用户点击"开始休息"后，才进入休息倒计时
- 休息结束后，系统提示用户可以开始下一轮
- 系统不会自动开始下一轮专注

---

### 7.5 手动下一轮

**功能说明：** 本产品不自动循环番茄钟，下一轮必须由用户手动开始。

**功能点：**
- 休息结束后显示"开始下一轮"按钮
- 用户可以选择继续当前任务
- 用户可以切换到其他任务
- 用户可以修改专注时间

---

### 7.6 托盘运行

**功能说明：** 软件需要支持最小化到系统托盘，避免占用任务栏空间。

**功能点：**
- 关闭按钮最小化到托盘（可配置为直接退出）
- 托盘图标展示当前状态
- 托盘右键菜单（显示主窗口、开始/暂停、停止摄像头提醒等）
- 专注计时在托盘状态下继续运行
- 最小化后显示灵动岛浮动倒计时

---

### 7.7 开机自启

**功能说明：** 用户可以在设置中开启或关闭开机自启。

- 默认关闭开机自启
- 通过注册表 `CurrentVersion\Run` 实现
- 开机启动后默认进入空闲状态，不自动开始番茄钟

---

### 7.8 学习任务管理

**功能说明：** 用户可以创建学习任务，并将每一轮番茄钟绑定到具体任务上。

**预置任务模板（首次启动自动创建）：**

| 分类 | 预设任务 |
|------|----------|
| 数学 | 考研数学 |
| 英语 | 考研英语 |
| 政治 | 考研政治 |
| 专业课 | 专业课 |
| 其他 | 复盘与整理 |

**功能点：**
- 用户首次启动时自动获得预置任务模板
- 支持创建自定义任务、选择分类和颜色标签
- 支持编辑、删除任务
- 开始专注前可以选择任务
- 完成番茄钟后记录任务归属

---

### 7.9 今日学习统计

**功能说明：** 系统需要记录用户每天完成了几个番茄钟，便于自我反馈。

**主界面显示：**
- 今日完成番茄钟数 / 今日目标番茄数
- 今日专注总时长
- 连续学习天数（Streak）

**统计页面（完整）：**
- 今日/周/月切换视图
- 热力图日历（30 天学习强度分布）
- 小时分布图（一天中什么时段效率最高）
- 任务分布饼图（各任务番茄钟占比）
- 周趋势图（每周专注时长变化）
- 效率趋势图（完课率、平均时长、平均质量）

---

### 7.10 专注质量系统

**功能说明：** 帮助用户评估每一轮专注的实际质量，而不仅仅是"完成了"。

**专注评分（1-5 星）：**
- 专注完成后，用户可以手动给这轮番茄钟打分
- 星星显示在完成面板中，一键点击即可评分
- 评分数据参与洞察分析和每日复盘报告

**专注笔记：**
- 专注完成后，用户可以快速记录备注（如"做到第X题"、"错题已标记"）
- 笔记存储在 session 数据中，后续可在统计页回顾
- 笔记上限 200 字

---

### 7.11 走神检测（Presence Detection）

**功能说明：** 利用摄像头检测用户是否离开座位，在专注期间提醒用户回到学习状态。

**行为：**
- 专注中，定期检测用户是否在摄像头前
- 检测到离开 → 发送系统通知"走神提醒"
- 连续检测到离开最多提醒 3 次（防止骚扰）
- 严重提醒强度下，走神检测触发时主窗口强制置顶
- 用户回到后重置计数器

**配置项：**
- 走神检测开关（默认开启）
- 检测间隔（默认 5 秒，可配置 3-30 秒）

---

### 7.12 洞察引擎（Insights）

**功能说明：** 基于历史专注数据，自动生成学习习惯洞察。

**洞察类型：**
| 类型 | 示例 |
|------|------|
| 峰值时段 | "你的专注高峰时段是上午 9-11 点" |
| 最优日 | "周三是你一周中效率最高的一天" |
| 趋势 | "本周专注时长比上周提升了 15%" |
| 连续学习 | "你已经连续专注 7 天，保持！" |
| 科目均衡 | "数学本周尚未学习，建议今天安排" |
| 里程碑 | "恭喜！累计完成 100 个番茄钟" |

**科目均衡分析：**
- 周视图下，如果某个科目学习时长为 0，显示预警
- 帮助考研用户防止偏科

---

### 7.13 每日复盘报告

**功能说明：** 每次启动软件时，展示昨天的学习回顾。

**报告内容：**
- 昨日完成番茄钟数
- 昨日专注总时长
- 主要学习科目
- 平均专注质量评分
- 连续学习天数
- 涉及科目数量

**展示规则：**
- 仅在当天首次启动且昨天有记录时弹出
- 同一天不会重复弹出
- 用户可以关闭此功能

---

### 7.14 考试倒计时

**功能说明：** 在设置中配置考试日期和考试名称，主界面底部显示倒计时天数。

- 默认考试名称：考研
- 仅在考试日期在未来时显示
- 天数每日自动更新

---

### 7.15 数据导出

**功能说明：** 支持导出学习数据，用于备份或外部分析。

- 导出格式：CSV / JSON
- 导出内容：专注记录、任务列表、统计摘要
- 导出路径由用户选择

## 8. 页面设计需求

### 8.1 主界面（TimerPage）

**页面元素：**
- 顶部：任务选择药丸（点击弹出任务切换菜单）
- 中央：圆环进度 + 大号倒计时数字（72pt Consolas）
- 操作区（按状态切换）：

## 9. 权限与隐私要求

### 9.1 摄像头权限

软件需要调用 Windows 摄像头权限。首次使用摄像头提醒时，如果系统未授权，需要引导用户开启权限。

### 9.2 隐私原则

- 不保存摄像头画面
- 不展示摄像头画面
- 不上传任何摄像头数据
- 不连接外部服务器
- 所有配置和数据保存在本地 `%APPDATA%/LumenPomodoro/`
- 摄像头仅用于触发硬件指示灯和走神检测

### 9.3 用户提示

首次启用摄像头提醒时，必须弹窗明确说明：

> 本软件仅在番茄钟结束或休息阶段根据你的设置调用摄像头，用于触发摄像头指示灯提醒。软件不会拍照、录像、保存或上传摄像头画面。

用户需点击"同意"后才启用摄像头提醒。

### 9.4 摄像头占用提示

当摄像头正在被软件占用时，界面必须明显展示：

```
摄像头提醒中：当前摄像头被用于点亮指示灯，不会保存或上传画面。
```

并提供"关闭摄像头提醒"按钮。

## 10. 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 摄像头权限未开启 | 提示用户开启 Windows 摄像头权限，自动打开隐私设置页 |
| 摄像头被其他软件占用 | 提示用户关闭占用摄像头的软件 |
| 摄像头不存在 | 提示未检测到可用摄像头 |
| 摄像头打开失败 | 使用弹窗 / 声音 / 系统通知作为兜底提醒 |
| 用户关闭所有提醒方式 | 提示"关闭所有提醒后，到点可能无法感知" |
| 软件最小化到托盘 | 计时继续运行，到点正常提醒 + 灵动岛 |
| 软件异常关闭 | 下次启动时恢复默认空闲状态，未完成的 session 自动保存 |
| 电脑睡眠 | 唤醒后根据当前时间重新计算状态，纠正计时器偏移 |
| 摄像头长时间未释放 | 30 分钟自动保护释放，并提示用户 |

### 12.1 实际技术栈

| 类别 | 技术 |
|------|------|
| 框架 | .NET 9 + WPF |

### 12.2 项目结构

```
LumenPomodoro/
├── Models/          # 数据模型（Settings, TaskItem, FocusSession, InsightModels）
├── ViewModels/      # MVVM ViewModel（Main, Settings, Stats, Tasks）
├── Views/           # XAML 页面和窗口
│   ├── Pages/       # TimerPage, SettingsPage, StatsPage, TasksPage
│   └── Dialogs/     # EditTaskDialog
├── Services/        # 业务逻辑
│   ├── Abstractions/ # 服务接口（IStorageService, ITimerService, ICameraService...）
│   ├── TimerService.cs
│   ├── CameraService.cs
│   ├── StorageService.cs
│   ├── SoundService.cs
│   ├── TrayService.cs
│   ├── InsightEngine.cs
│   ├── PresenceDetector.cs
│   └── ExportService.cs
├── Controls/        # 自定义控件（ArcProgress, HeatmapCalendar, Charts...）
├── Converters/      # WPF 值转换器
├── Interop/         # MF 互操作
├── Themes/          # 自定义样式
└── Fonts/           # Inter 字体子集
```

### 12.3 摄像头调用

使用 Windows Media Foundation 原生调用：
- `IMFMediaSource` 打开摄像头设备
- 读取视频流以保持设备激活
- 不显示画面、不保存帧、不上传数据
- 到达关闭条件后释放设备

### 12.4 发布结构

```
LumenPomodoro.zip
├── LumenPomodoro.exe      # 原生启动器（检测 .NET 9 Runtime，缺失时自动下载安装）
└── app/
    └── LumenPomodoro.exe  # WPF 主程序
```

发布包控制在约 16MB，不打包完整 .NET Desktop Runtime。
