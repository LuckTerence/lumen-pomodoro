## 1. 文档信息

| 项目 | 内容 |
|------|------|
| 产品名称 | 摄像头提醒番茄钟 |
| 产品定位 | 面向个人考研学习的 Windows 桌面专注计时工具 |
| 运行平台 | Windows 10 / Windows 11 |
| 文档类型 | PRD 产品需求文档 |
| 当前版本 | V1.1 |
| 目标用户 | 用户本人，主要用于考研复习、自律学习、专注提醒 |
| 核心卖点 | 番茄钟到点后调用笔记本摄像头，使摄像头指示灯持续亮起，用视觉方式提醒用户休息 |
| 推荐技术方向 | Windows 原生桌面应用，优先 C# + .NET + WinUI 3 / WPF |

## 2. 产品背景

用户在 Windows 笔记本上进行考研复习时，需要长时间专注学习，例如数学、英语、政治、专业课等。传统番茄钟通常依赖声音、弹窗或系统通知，但这些提醒方式容易被忽略：

- 声音可能被静音。
- 弹窗可能被窗口遮挡。
- 系统通知可能被勿扰模式拦截。
- 手机提醒容易让用户顺手刷手机，最后休息变成沉迷。

本产品利用笔记本摄像头指示灯通常会在摄像头被调用时自动亮起的硬件特性，在专注时间结束后调用摄像头，让摄像头灯持续亮起，形成更醒目、更难忽略的视觉提醒。

产品不保存照片、不录像、不上传摄像头数据，仅在本地调用摄像头并在用户确认或流程结束后释放摄像头。

## 3. 产品目标

### 3.1 核心目标

- 提供一个运行流畅、界面高级、适合长期使用的 Windows 番茄钟工具。
- 支持考研复习场景下的任务分类与学习统计。
- 在专注结束后，通过摄像头指示灯持续提醒用户该休息。
- 支持弹窗、声音、系统通知等辅助提醒，并允许用户单独开关。
- 摄像头亮灯方式支持用户自由选择，不固定为某一种模式。
- 不自动进入休息，不自动循环，所有关键流程由用户手动确认。
- 支持托盘运行和可选开机自启。
- 保证摄像头调用本地化、无存储、无上传。

### 3.2 非目标

- 不做用户登录。
- 不做云同步。
- 不做人脸识别。
- 不拍照、不录像。
- 不上传任何摄像头画面。
- 不做团队协作。
- 不做复杂社交、排行榜、打卡社区。
- 不做过度复杂的任务管理系统。

## 4. 用户定位

### 4.1 主要用户

本软件主要供用户本人使用，核心场景是考研复习。

### 4.2 用户特点

- 使用 Windows 笔记本学习或办公。
- 希望通过番茄钟管理专注时间。
- 容易忽略普通提醒。

- 需要按考研科目记录专注任务。
- 希望软件界面简约、高级、耐看。

## 6. 核心用户流程

### 6.1 专注流程

```
选择任务 / 科目
↓
设置或使用默认专注时间
↓
点击"开始专注"
↓
进入专注倒计时
↓
倒计时结束
↓
触发摄像头灯提醒 + 可选弹窗 / 声音 / 系统通知
↓
用户手动确认或处理提醒
↓
用户手动选择是否进入休息倒计时
```

### 6.2 休息流程

```
专注结束
↓
用户点击"开始休息"
↓
进入休息倒计时
↓
休息结束
↓
提示用户可以开始下一轮专注
↓
用户手动开始下一轮
```

### 6.3 不自动流程原则

本产品不自动进入休息倒计时，也不自动开启下一轮番茄钟。

原因：

- 用户可能刚好还在收尾一道题。
- 用户可能需要先记录错题或标注进度。
- 用户可能需要决定是否继续同一科目。
- 考研场景不适合完全自动循环，容易和真实学习节奏冲突。

## 7. 核心功能需求

### 7.1 番茄钟计时

**功能说明：** 用户可以启动一个专注倒计时，用于考研学习。

**功能点：**

- 默认专注时间：25 分钟
- 支持自定义专注时间
- 支持开始、暂停、继续、重置计时
- 倒计时过程中展示剩余时间
- 当前专注任务需要和本轮番茄钟绑定

**验收标准：**

- 用户点击"开始专注"后，倒计时每秒递减
- 用户点击"暂停"后，倒计时停止
- 用户点击"继续"后，从暂停时间继续倒计时
- 用户点击"重置"后，倒计时恢复默认或当前设置时间
- 专注完成后，系统记录本轮完成数据

---

### 7.2 摄像头灯提醒

**功能说明：** 当专注时间结束后，程序调用本机摄像头，使摄像头指示灯亮起，用于提醒用户休息。

#### 亮灯模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 固定时长模式 | 摄像头灯持续亮起指定时长 | 用户只需要明显提醒，不希望长时间占用摄像头 |
| 直到确认模式 | 摄像头灯持续亮起，直到用户点击确认 | 防止用户错过提醒 |
| 跟随休息模式 | 摄像头灯在休息倒计时期间保持亮起 | 用户希望摄像头灯作为完整休息状态提示 |

#### 直到确认模式规则

```
专注倒计时结束
↓
摄像头灯立即亮起
↓
持续亮灯
↓
用户点击"我知道了"或"关闭提醒"
↓
摄像头释放，灯关闭
```

- 直到确认模式下，用户点击"开始休息"视为已确认提醒
- 点击后关闭本次提醒摄像头
- 是否在休息期间继续亮灯，由"休息期间是否亮灯"设置决定

#### 固定时长模式规则

```
专注倒计时结束
↓
摄像头灯立即亮起
↓
持续用户设置的时长
↓
到时自动关闭摄像头
```

固定时长可配置：30秒、1分钟、3分钟、5分钟、10分钟、自定义

#### 跟随休息模式规则

```
专注倒计时结束
↓
弹出完成状态
↓
用户点击"开始休息"
↓
系统进入休息倒计时
↓
摄像头灯开始亮起，并跟随整个休息阶段
↓
休息倒计时结束
↓
系统自动释放摄像头，摄像头灯关闭
```

#### 摄像头提醒配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 摄像头提醒开关 | Boolean | 开启 | 是否启用摄像头灯提醒 |
| 摄像头提醒模式 | Enum | 直到确认模式 | 固定时长 / 直到确认 / 跟随休息 |
| 固定亮灯时长 | Number | 3 分钟 | 固定时长模式下生效 |
| 休息期间是否亮灯 | Boolean | 开启 | 跟随休息模式或休息阶段联动时生效 |
| 摄像头编号 | Number | 0 | 默认摄像头索引 |
| 提醒中允许手动关闭 | Boolean | 开启 | 防止摄像头长时间占用 |

#### 功能点

- 到点后根据用户选择的模式调用摄像头
- 摄像头提醒模式必须可在设置中修改
- 摄像头亮灯持续时间必须可配置
- 摄像头提醒期间，界面应展示当前提醒状态
- 用户可以手动停止摄像头提醒
- 摄像头调用结束后必须自动释放摄像头
- 摄像头打开失败时，需要给出错误提示
- 软件不得保存、展示、上传摄像头画面

---

### 7.3 到点提醒

**功能说明：** 番茄钟结束时，除了摄像头灯提醒，也应提供基础提醒方式，避免摄像头不可用时完全无提示。

| 提醒方式 | 默认状态 | 是否可配置 | 说明 |
|----------|----------|------------|------|
| 摄像头灯提醒 | 开启 | 是 | 核心提醒方式 |
| 弹窗提醒 | 开启 | 是 | 到点后弹出确认窗口 |
| 声音提醒 | 开启 | 是 | 播放提示音 |
| 系统通知 | 开启 | 是 | 使用 Windows 通知中心提醒 |

**验收标准：**

- 用户可以独立开启或关闭每一种提醒方式
- 到点后至少启用一种提醒方式
- 如果用户关闭所有提醒方式，系统需要提示风险
- 摄像头打开失败时，仍然使用弹窗、声音或系统通知兜底

---

### 7.4 手动休息倒计时

**功能说明：** 专注结束后，系统不自动进入休息阶段。用户需要点击"开始休息"后才进入休息倒计时。

**功能点：**

- 默认短休息时间：5 分钟
- 默认长休息时间：15 分钟
- 支持用户手动开始休息
- 支持用户跳过休息
- 支持用户提前结束休息
- 支持休息结束提醒
- 支持休息阶段是否联动摄像头灯

**验收标准：**

- 专注结束后，系统停留在"专注完成"状态
- 用户点击"开始休息"后，才进入休息倒计时
- 休息结束后，系统提示用户可以开始下一轮
- 系统不会自动开始下一轮专注

---

### 7.5 手动下一轮

**功能说明：** 本产品不自动循环番茄钟，下一轮必须由用户手动开始。

**功能点：**

- 休息结束后显示"开始下一轮"按钮
- 用户可以选择继续当前任务
- 用户可以切换到其他任务
- 用户可以修改专注时间

**验收标准：**

- 休息结束后，不自动启动下一轮
- 用户必须点击"开始专注"才进入下一轮

---

### 7.6 托盘运行

**功能说明：** 软件需要支持最小化到系统托盘，避免占用任务栏空间。

**功能点：**

- 点击关闭按钮时，可选择最小化到托盘
- 支持托盘图标展示当前状态
- 支持托盘右键菜单
- 支持从托盘恢复主窗口
- 专注计时在托盘状态下继续运行

**托盘右键菜单：**

| 菜单项 | 功能 |
|--------|------|
| 显示主窗口 | 恢复主界面 |
| 开始 / 暂停 | 快速控制当前计时 |
| 停止摄像头提醒 | 摄像头亮灯时可快速关闭 |
| 今日统计 | 简要显示今日完成番茄钟数量 |
| 设置 | 打开设置页面 |
| 退出 | 完全退出软件 |

**验收标准：**

- 最小化到托盘后，倒计时仍然准确运行
- 到点后仍然可以触发提醒
- 摄像头提醒中，托盘菜单可以关闭摄像头

---

### 7.7 开机自启

**功能说明：** 用户可以在设置中开启或关闭开机自启。

**功能点：**

- 默认关闭开机自启
- 用户可在设置中开启/关闭
- 开机启动后默认进入空闲状态，不自动开始番茄钟

**验收标准：**

- 开启后，Windows 登录时自动启动软件
- 关闭后，Windows 登录时不自动启动软件
- 开机启动不应自动开始计时

---

### 7.8 学习任务管理

**功能说明：** 用户可以创建考研学习任务，并将每一轮番茄钟绑定到具体任务上。

**默认任务分类：**

| 分类 | 示例任务 |
|------|----------|
| 数学 | 高数、线代、概率论、刷题、错题复盘 |
| 英语 | 单词、阅读、翻译、作文、真题复盘 |
| 政治 | 马原、史纲、毛中特、思修、选择题 |
| 专业课 | 数据结构、计算机网络、操作系统、组成原理等 |
| 其他 | 复盘、计划、背诵、整理资料 |

**功能点：**

- 支持创建自定义任务
- 支持编辑任务名称
- 支持删除任务
- 支持任务分类
- 开始专注前可以选择任务
- 完成番茄钟后记录任务归属

---

### 7.9 今日学习统计

**功能说明：** 系统需要记录用户每天完成了几个番茄钟，便于考研复习自我反馈。

**统计指标：**

| 指标 | 说明 |
|------|------|
| 今日完成番茄钟数 | 当天完成的专注轮数 |
| 今日专注总时长 | 当天完成的专注分钟数 |
| 按任务统计 | 每个任务完成了多少轮 |
| 当前连续专注轮数 | 当天连续完成的番茄钟数量 |
| 本周累计专注时长 | 可选增强功能 |

**验收标准：**

- 专注倒计时自然完成后，记录一轮完成
- 用户手动重置或中途退出，不计入完成
- 今日统计在主界面可见
- 历史统计可在统计页查看

## 9. 权限与隐私要求

### 9.1 摄像头权限

软件需要调用 Windows 摄像头权限。首次使用摄像头提醒时，如果系统未授权，需要引导用户开启权限。

### 9.2 隐私原则

- 不保存摄像头画面
- 不展示摄像头画面
- 不上传任何摄像头数据
- 不连接外部服务器
- 所有配置保存在本地
- 摄像头仅用于触发硬件指示灯

### 9.3 用户提示

首次启用摄像头提醒时，应明确说明：

> 本软件仅在番茄钟结束或休息阶段根据你的设置调用摄像头，用于触发摄像头指示灯提醒。软件不会拍照、录像、保存或上传摄像头画面。

### 9.4 摄像头占用提示

当摄像头正在被软件占用时，界面必须明显展示：

```
摄像头提醒中：当前摄像头被用于点亮指示灯，不会保存或上传画面。
```

并提供"关闭摄像头提醒"按钮。

## 10. 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 摄像头权限未开启 | 提示用户开启 Windows 摄像头权限 |
| 摄像头被其他软件占用 | 提示用户关闭占用摄像头的软件 |
| 摄像头不存在 | 提示未检测到可用摄像头 |
| 摄像头打开失败 | 使用弹窗 / 声音 / 系统通知作为兜底提醒 |
| 用户关闭所有提醒方式 | 提示"关闭所有提醒后，到点可能无法感知" |
| 软件最小化到托盘 | 计时继续运行，到点正常提醒 |
| 软件异常关闭 | 下次启动时恢复默认空闲状态 |
| 电脑睡眠 | 唤醒后根据当前时间重新计算状态 |
| 摄像头长时间未释放 | 自动保护释放，并提示用户 |

## 12. 技术实现建议

### 12.1 技术选型原则

用户要求性能必须好，并倾向原生性能。

| 方案 | 推荐程度 | 说明 |
|------|----------|------|
| C# + WinUI 3 | 高 | 更现代的 Windows 原生界面，适合 Fluent / Acrylic / Mica 风格 |
| C# + WPF | 高 | 成熟稳定，生态完善，性能好，适合快速开发桌面工具 |
| C++ + WinUI / Qt | 中 | 性能强，但开发成本更高 |
| Python + PySide6 | 中 | 开发快，但原生质感和打包体验略弱 |
| Electron | 低 | 资源占用偏高 |
| Tkinter | 低 | 界面质感不符合当前产品要求 |

**建议结论：**

优先选择：`C# + .NET + WinUI 3`
备选选择：`C# + WPF`

### 12.2 摄像头调用

摄像头调用只需要打开设备流，不需要展示画面。

**实现原则：**

- 打开摄像头设备
- 读取视频流以保持设备激活
- 不显示画面、不保存帧、不上传数据
- 到达关闭条件后释放设备

| 技术 | 说明 |
|------|------|
| Windows MediaCapture | 更原生，适合 C# / WinUI |
| DirectShow | 较底层，兼容性好 |
| OpenCVSharp | 实现简单，但会引入额外依赖 |

推荐优先考虑 Windows 原生 MediaCapture 或 DirectShow。

### 12.3 托盘实现

- WPF 可使用 NotifyIcon 方案
- WinUI 3 需要结合 Windows App SDK 或第三方托盘实现方式
- 托盘菜单必须支持"停止摄像头提醒"

### 12.4 开机自启实现

- 写入 Windows 启动项
- 使用启动文件夹快捷方式
- MSIX 打包时使用 StartupTask
